以下项目全用python实现，要求自动命名模块文件名。

模块：
- **模块一**：读取本机的开机时间，计算本机运行时间。
- **模块二**：通过`ping`功能检测本机是否连通互联网。
- **模块三**：检测指定挂载分区的使用情况，包含百分比数据计算。
- **模块四**：编写一个以send.py为基础的邮件发信模块。
- **模块五**：用来格式化并编写邮件标题和邮件正文后可通过**模块四**发件。要求发件的邮件标题在半个小时之内不能相同（用来规避因短时间发送邮件标题相同而被屏蔽），假如需要在半小时内发送多个相同的邮件标题，则在邮件标题开头随机添加不同的内容如：`哦齁齁❤`/`啊齁齁❤`/`齁唔❤`/`去了去了❤`/`嗯？❤`。

主程序：
编写一个这样的主程序xiaoU.py：
第一个行为：主程序在启动后会先持续调用**模块二**检测是否联网，假如首次发现联网成功则调用**模块一**获取开机时间，并将开机时间及运行时间内容用**模块五**编辑邮件内容，并将邮件标题设定为 __“小悠上线提醒”__，然后将邮件通过**模块四**发送。要求该行为的联网检测并发件只会存在首次，如果持续断网或后续断网则不执行发件并持续检测，直到发现联网后发件。
第二个行为：主程序在调用**模块二**后，会持续调用**模块三**检测挂载分区使用情况，如果发现指定的分区剩余容量低于90GB则调用**模块五**编辑邮件内容，并将邮件标题设定为 __“小悠提醒你空间不够了！”__，然后通过**模块四**发送邮件。该行为持续检测挂载分区情况，但发件行为间隔为180分钟一次。

为该项目提供requirements.txt文件，并且该项目可以通过python的venv环境部署运行。

xiaoU.py在原来的基础上，添加一项行为：持续检测联网并在首次发现联网发件后，假如持续检测中发现联网中断，则在等待重新联网后调用**模块五** **email_composer**编辑邮件内容，并将邮件标题设定为 __“小悠已重新联网”__，然后将邮件通过**模块四**发送。
